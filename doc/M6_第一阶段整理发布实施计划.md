# M6 第一阶段整理发布实施计划

## 1. 目标

M6 的目标是把第一阶段从“功能可跑”整理成“可复用、可交付、可验证”的版本：

- 固化 README 和使用说明。
- 明确 Web Viewer 与 Qt Viewer 的阶段边界。
- 记录板级验证结果。
- 提供命令行串口验证工具。
- 为后续 v0.1 tag/release 做准备。

## 2. 当前新增内容

- `README.md`
- `doc/OpenFPGA_Debug_Core_使用说明.md`
- `doc/OpenFPGA_Web_Viewer_使用说明.md`
- `doc/OpenFPGA_Studio_板级验证记录.md`
- `tools/viewer/serial_validate.ps1`

## 3. 第一阶段当前完成度

已完成：

- Debug Protocol v1 文档。
- RTL Debug Core、timestamp、ring buffer、packetizer、UART TX。
- Event/Watch/Debug Print/Heartbeat/Status 输出。
- Web Viewer 基础视图、暂停、清空、CSV/JSONL 导出。
- KU5P 板级 demo、XDC、bitstream、COM6 实测。

阶段性调整：

- 原规划写的是 Qt Viewer。当前第一阶段以 Web Viewer 作为可验收实现，Qt Viewer 后续再实现。

待完成：

- 30 分钟长稳记录。
- v0.1 tag 或 release 包。
- Qt Viewer。
- Watch 曲线、文本映射、关键字过滤等增强项。

## 4. 发布前检查清单

- 仿真：
  - `tb_openfpga_debug_protocol` PASS。
  - `tb_openfpga_debug_m3` PASS。
  - `tb_openfpga_debug_board_demo` PASS。
- PC parser：
  - `python tools\viewer\protocol_parser_test.py` PASS。
- Vivado：
  - `prj/scripts/check_openfpga_debug_m5_elab.tcl` PASS。
  - bitstream 可生成。
- 板级：
  - Web Viewer 能连接板卡串口。
  - `serial_validate.ps1` 30 分钟无 checksum error。
  - `drop_count` 不持续增长。

## 5. 后续建议

v0.1 发布后，建议按优先级推进：

1. 补 Qt Viewer 或把 Web Viewer 正式定位为第一版默认 Viewer。
2. 增加 `print_id/event_id/watch_id` 的文本映射配置。
3. 增加 921600/1000000 baud 板级验证。
4. 增加 PC 到 FPGA 的控制通道设计。
5. 为长时间大数据场景优化 Viewer 渲染。
