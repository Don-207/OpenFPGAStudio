# M5 板级集成验证实施计划

## 1. 目标

M5 将 M3/M4 已完成的 Debug Core 与 Viewer 闭环接入当前 `xcku5p-ffvb676-2-i` Vivado 工程，形成一个可下载到真实板卡的最小 demo：

- Debug Core 周期性输出 `HEARTBEAT/STATUS`。
- Demo 逻辑周期性输出 `EVENT/WATCH/DEBUG_PRINT`。
- UART TX 连接到板卡串口，Web Viewer 可实时解析五类协议帧。
- 两个 LED 输出暴露 heartbeat 与 event/drop 状态，方便无上位机时快速判断运行状态。

## 2. RTL 变更

### 2.1 `openfpga_debug_core.v` / `openfpga_debug_top.v`

新增两个输入：

- `heartbeat_valid`：封装 `OFD_TYPE_HEARTBEAT`，payload 为 `u32 timestamp`。
- `status_valid`：封装 `OFD_TYPE_STATUS`，payload 为 `u32 timestamp + u16 buffer_used + u16 drop_count + u16 packet_count`。

输入仲裁优先级为：

```text
event > watch > debug_print > heartbeat > status
```

同周期低优先级消息不会进入 ring buffer，并计入 `drop_count`。板级 demo 自身使用 pending 标志错峰发送，避免正常周期消息互相挤掉。

### 2.2 `rtl/board/openfpga_debug_board_demo.v`

新增板级 demo 顶层，端口为：

```text
clk_p
clk_n
reset_n
demo_trigger
uart_tx
led0
led1
```

`led0` 为 heartbeat 闪烁；`led1` 平时为 event 脉冲，`drop_count` 非零后常亮报警。

默认参数：

- `CLK_FREQ_HZ = 100000000`
- `UART_BAUD = 115200`
- heartbeat 每 1 秒一次
- status 每 20 ms 一次
- watch 每 10 ms 一次
- event 每 100 ms 一次
- debug print 每 200 ms 一次

`demo_trigger` 上升沿会额外触发一条 event 和一条 debug print，用于板上按钮或外部信号验证。

## 3. Vivado 集成

新增：

- `prj/scripts/integrate_openfpga_debug_m5.tcl`
- `prj/constraints/openfpga_debug_board_demo.xdc`

在 Vivado Tcl Console 中执行：

```tcl
source prj/scripts/integrate_openfpga_debug_m5.tcl
```

脚本会：

- 打开 `prj/OpenFPGAStudio.xpr`。
- 将 Debug Core RTL 与 M5 board demo 加入 `sources_1`。
- 设置 include 路径为 `rtl/openfpga_debug`。
- 设置综合顶层为 `openfpga_debug_board_demo`。
- 加入 XDC 并更新工程编译顺序。

当前 `openfpga_debug_board_demo.xdc` 已根据厂家 `pin.xdc` 固化 KU5P 板级管脚和 IOSTANDARD：

```text
clk_p        J23  DIFF_HSTL_I_12
clk_n        J24  DIFF_HSTL_I_12
reset_n      L23  LVCMOS18
demo_trigger N22  LVCMOS18
uart_tx      F15  LVCMOS18
led0         G26  LVCMOS18
led1         G25  LVCMOS18
```

## 4. 验收仿真

新增 `sim/board/tb_openfpga_debug_board_demo.v`，从 demo 的 `uart_tx` 线上反采样 UART 字节并检查：

- 至少收到一帧 `HEARTBEAT`。
- 至少收到一帧 `EVENT`。
- 至少收到一帧 `WATCH`。
- 至少收到一帧 `DEBUG_PRINT`。
- 至少收到一帧 `STATUS`。
- 帧头、版本、长度和 XOR checksum 正确。

仿真命令：

```powershell
xvlog -i rtl\openfpga_debug rtl\openfpga_debug\openfpga_debug_pkg.vh rtl\openfpga_debug\openfpga_debug_timestamp.v rtl\openfpga_debug\openfpga_debug_ring_buffer.v rtl\openfpga_debug\openfpga_debug_packetizer.v rtl\openfpga_debug\openfpga_debug_uart_tx.v rtl\openfpga_debug\openfpga_debug_core.v rtl\openfpga_debug\openfpga_debug_top.v rtl\board\openfpga_debug_board_demo.v sim\board\tb_openfpga_debug_board_demo.v
xelab tb_openfpga_debug_board_demo -s tb_openfpga_debug_board_demo_sim
xsim tb_openfpga_debug_board_demo_sim -runall
```

期望结果：

```text
PASS: OpenFPGA Debug M5 board demo emitted heartbeat/event/watch/print/status frames
```

## 5. 板级验证流程

1. 在 Vivado 中执行 M5 Tcl 集成脚本。
2. 确认 `prj/constraints/openfpga_debug_board_demo.xdc` 与当前板卡一致。
3. 综合、实现并生成 bitstream。
4. 下载 bitstream 后，将 UART TX 接到 PC 串口 RX，共地。
5. 打开 `tools/viewer/web/index.html`，选择与 RTL 一致的 baud rate，默认 `115200`。
6. 确认 Viewer 持续收到 heartbeat/status。
7. 触发 `demo_trigger`，确认 Viewer 中实时出现 event/debug print。
8. 连续运行 30 分钟，确认无解析失步、Viewer 无崩溃，`drop_count` 不持续增长。
