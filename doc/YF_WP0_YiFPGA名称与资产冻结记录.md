## 5. 可审计名称基线

执行：

```bash
python3 tools/branding/audit_brand_names.py --summary
```

扫描器读取 Git 跟踪文件及未忽略的新文件，逐行识别 `OpenFPGA`、`openfpga`、`OPENFPGA`
和 `OFD_`，
分类为：

- `brand_text`：用户可见品牌与一般文档文本；
- `source_identifier`：源码、脚本和测试中的旧标识；
- `protocol_abi`：冻结 schema、诊断类型及 `OFD_*` 常量；
- `historical_evidence`：历史计划、验证、验收和归档记录；
- `generated`：受版本控制的厂商生成路径（如存在）。

完整 JSON（不加 `--summary`）包含文件、行号和匹配词，可供各工作包生成差异。分类用于
迁移决策，不表示允许永久保留；ABI 与历史证据是保留项，其余分类应随工作包单调收敛。
