# M26 板级验证与第五阶段发布实施计划

M26 是第五阶段 Open Logic Analyzer 的集成和发布里程碑。目标是在真实 board demo 上完成从 bitstream、串口链路、Viewer 波形显示到导出的验收闭环，并把验证记录、使用说明和发布 checklist 收口。

## 1. 目标

- 生成包含 Logic Analyzer 的 board demo bitstream。
- 在真实串口链路上完成 arm、trigger、readout、waveform display 和 export 验收。
- 连续运行 Debug/Trace/Monitor/Profiler/Logic Analyzer 共存场景，确认控制响应和数据上传稳定。
- 完成验证记录、发布 checklist 和使用说明。

## 6. 板级验收流程

使用 Chrome 或 Edge 打开 Web Viewer：

1. 下载包含 Logic Analyzer 的 board demo bitstream。
2. 连接 FPGA UART TX/RX，并选择与 RTL 一致的 baud rate。
3. 在 Monitor 视图读取 `LA_ID/LA_VERSION`。
4. 设置 `LA_SAMPLE_DIVISOR`、`LA_CAPTURE_DEPTH`、`LA_PRETRIGGER_DEPTH`。
5. 配置 trigger channel/value/mask，例如捕获 `debug_tx_valid` 上升沿。
6. 写 `LA_COMMAND.arm`，确认 `LA_STATUS.state = ARMED`。
7. 触发 demo 事件，确认收到 `LA_TRIGGER_EVENT` 且 capture done。
8. 写 `LA_COMMAND.start_readout` 或启用 auto_readout，等待 chunks 上传完成。
9. 切换到 Logic Analyzer 视图，确认波形、触发线和通道名正确。
10. 导出 VCD/JSONL，确认文件包含本次 capture。
11. 执行 stop/clear/再次 arm，确认 capture_id 递增且旧数据不污染新捕获。

## 7. 长稳场景

连续运行 30 分钟：

- Debug status/heartbeat 持续可见。
- Trace 事件正常进入 Viewer。
- Monitor 读写 LA 和 Profiler 寄存器响应稳定。
- Profiler snapshot 持续更新。
- LA 周期性 arm、trigger、readout、clear。
- checksum error、dropped frames、overflow 不持续增长。
- UART 链路断开重连后 Viewer 不崩溃，重新注入或重新连接后可恢复显示。

## 9. 已知限制

第五阶段 P0 发布说明必须明确：

- P0 不是完整替代 Vivado ILA 或 SignalTap。
- P0 默认单采样时钟域，跨域信号需先同步。
- P0 默认 sample width 不超过 32 bit。
- P0 默认有限 depth，推荐 64 或 128 samples 起步。
- P0 采用捕获后分片读出，不做全带宽连续流式上传。
- P0 不做压缩波形和复杂序列触发。
